# development

developmentはハーネス運用を行う部署です。
Planner・Generator・Evaluatorの3エージェントが連携し、仕様の分解から実装・検証までを担います。

## 役割

- PMから受け取った要求を仕様とタスクに分解する
- スプリント単位で実装を進め、自己評価と外部検証を行う
- 完成物をQAに渡す前に動作確認を済ませる
- 技術的なリスクやブロッカーをPMに報告する

## 入力

- PMからの依頼票（機能仕様・期限・成果物の定義）
- analysisまたは社長からの技術選定相談

## 出力

- 実装済みコード・設定ファイル
- 動作確認済みの成果物
- 実装メモ（設計判断・既知の制約・注意点）
- 技術的ブロッカーのPMへの報告

## ルール

- 仕様が曖昧な場合は着手前にPMに確認する
- 勝手に仕様を広げない
- 技術的負債を発生させた場合は必ず記録する
- セキュリティ・パフォーマンスに影響する判断は単独で行わない

## 担当AI

Claude Code主担当 + Codex副担当

## AIの使い方

- 実装・リファクタリング・コードレビューはClaude Codeを主に使う
- 補完候補の生成やスニペット作成にはCodexを補助的に使う
- AIが生成したコードは必ず人間がレビューしてからコミットする
- セキュリティ・設計判断・仕様解釈はAIに委ねず人間が確認する
