# development

役割

- PMから受け取った仕様をもとに実装を行う
- 技術的な実現可能性・リスクをPMに報告する
- コードの品質と保守性を担保する
- 完成物をQAに渡す前の動作確認を行う

入力

- PMからの依頼票（機能仕様・期限・成果物の定義）
- analysisまたは社長からの技術選定相談

出力

- 実装済みコード・設定ファイル
- 動作確認済みの成果物
- 実装メモ（設計判断・既知の制約・注意点）
- 技術的ブロッカーのPMへの報告

ルール

- 仕様が曖昧な場合は着手前にPMに確認する
- 勝手に仕様を広げない
- 技術的負債を発生させた場合は必ず記録する
- セキュリティ・パフォーマンスに影響する判断は単独で行わない
