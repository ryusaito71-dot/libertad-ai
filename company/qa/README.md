# qa

役割

- 納品・公開・提出前の成果物を検査する
- 仕様・要件との整合性を確認する
- 不備・バグ・抜け漏れを検出してdevelopmentまたは担当部署に差し戻す
- 合格基準を満たした成果物にのみ承認を出す

入力

- developmentからの実装済み成果物
- PMからの検査依頼（合格基準・チェック観点・期限）
- 仕様書・要件定義書・デザイン仕様

出力

- QAレポート（合格・差し戻し・保留の判定と根拠）
- 不具合リスト（再現手順・深刻度・担当部署）
- 承認済み成果物の引き渡し通知

ルール

- 合格基準が定義されていない成果物は検査を開始しない
- 判定は根拠とともに記録する
- 差し戻しは担当部署が再現できる形で記述する
- QAは実装を修正しない。修正はdevelopmentに戻す

## 担当AI

ChatGPT + Claude Code + Codex

## AIの使い方

- テストケース・チェックリストの生成にChatGPTを使う
- コードレビュー・静的解析補助にClaude Codeを使う
- テストコードのスニペット生成にCodexを補助的に使う
- AI検査の結果はそのまま合格判定にせず、必ず人間が最終承認を行う
- 差し戻し判定・合格判定の根拠は人間が確認・記録する
