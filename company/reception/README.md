# reception（受付部署）

## 役割

- 外部からの依頼を最初に受け取る
- 依頼内容を構造化して社内フォーマットに変換する
- 秘書（secretary）へ転送し、社内フローを起動する
- 受付番号を採番してトレーサビリティを確保する

## 入力

- 依頼者からの自由記述（依頼内容、目的、欲しい成果物、制約条件）

## 出力

- `REQUESTS/CASE_XXX.md`：受付票（1件1ファイル）
- `company/secretary/INBOX.md` への追記
- `company/secretary/REQUEST_TEMPLATE.md` への反映

## ルール

- 依頼は必ず構造化してから次部署へ渡す（生テキストで流さない）
- 受付番号は3桁ゼロ埋め（001, 002, ...）で連番採番する
- 受付票には受付日時・ステータス・次渡し先を必ず記載する
- 内容の判断・編集は行わない（受け取って記録して流すのみ）
- DAILY_LOG・TODAY は今フェーズでは触らない

## ファイル構成

```
company/reception/
├── README.md          # この文書
└── REQUESTS/          # 受付票（1件1ファイル）
    ├── CASE_001.md
    ├── CASE_002.md
    └── ...
```
